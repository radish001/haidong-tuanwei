## MODIFIED Requirements

### Requirement: 企业信息表单维护
系统必须允许管理员通过表单流程对企业信息执行新增、编辑、查看、查询和删除操作，并以更接近目标设计稿的列表布局和抽屉化表单视觉呈现企业维护流程。

#### Scenario: 新增企业信息
- **WHEN** 管理员提交包含必填企业字段的合法企业表单
- **THEN** 系统保存企业信息，并在企业列表中展示该记录

#### Scenario: 编辑企业信息
- **WHEN** 管理员修改一条已存在的企业信息并提交合法数据
- **THEN** 系统保存修改结果，并展示更新后的企业信息

#### Scenario: 打开企业新增或编辑页面
- **WHEN** 管理员进入企业信息的新增或编辑页面
- **THEN** 系统以更接近右侧抽屉面板的视觉布局展示表单字段、操作按钮和辅助信息，同时保持现有页面路由和提交方式不变

### Requirement: 企业信息仅支持表单录入
系统必须提供企业列表筛选能力，并且第一阶段不得要求或提供企业 Excel 导入能力；企业列表页必须以统一视觉风格呈现筛选栏、操作按钮区和企业信息表格。

#### Scenario: 按条件筛选企业列表
- **WHEN** 管理员通过企业名称、行业等支持字段查询企业信息
- **THEN** 系统返回符合条件的企业记录，并以统一风格刷新企业列表

#### Scenario: 企业模块不提供导入入口
- **WHEN** 管理员进入企业信息维护页面
- **THEN** 系统仅提供表单新增和编辑流程，不提供 Excel 导入入口
