## ADDED Requirements

### Requirement: 青年信息导入失败数据下载
系统 MUST 在青年信息 Excel 导入存在失败行时，提供失败数据 Excel 下载能力。失败数据文件 MUST 仅包含导入失败的原始数据行，并复用当前青年信息导入模板的主工作表字段顺序、隐藏数据工作表和下拉选项来源。失败数据文件 MUST 在末尾增加 `失败原因` 列，用于记录每条失败数据对应的导入错误原因。

#### Scenario: 下载失败数据文件
- **WHEN** 管理员上传青年信息 Excel 且部分数据导入失败
- **THEN** 系统 MUST 提供一个失败数据 Excel 下载入口
- **AND** 下载文件 MUST 仅包含失败的原始数据行
- **AND** 下载文件 MUST 在末尾追加 `失败原因` 列

#### Scenario: 失败数据文件复用模板选项
- **WHEN** 管理员打开失败数据 Excel 文件
- **THEN** 文件前置业务列 MUST 保持与标准导入模板一致的字段顺序
- **AND** 性别、民族、学历、学校、学校所在区域、专业等受控字段 MUST 继续提供与标准导入模板一致的下拉选项
- **AND** `失败原因` 列 MUST 展示该行最近一次导入失败的原因说明

## MODIFIED Requirements

### Requirement: 青年信息 Excel 导入校验
系统 MUST 在保存前逐行校验上传的青年信息 Excel 数据，并按照当前字典、学校、专业、学历和区域主数据规则拦截非法值或重复数据。系统 MUST 对招考年份执行年份格式校验，只允许合法的 4 位年份值。由于导入模板采用基础采集字段结构，系统 MUST 不再要求模板中提供政治面貌、学位、毕业时间和就业方向列。系统 MUST 同时兼容标准 11 列导入模板与末尾附带 `失败原因` 列的失败数据返修文件，并在导入时忽略 `失败原因` 列内容。

#### Scenario: 导入合法青年数据
- **WHEN** 管理员上传按模板正确填写且数据合法的 Excel 文件
- **THEN** 系统将这些记录保存到统一青年信息表中，并写入当前分类对应的 `youth_type`
- **AND** 系统 MUST 为学历、学校和专业写入对应的官方编码标识
- **AND** 系统 MUST 为招考年份写入对应的年份值

#### Scenario: 区域字段按完整路径导入
- **WHEN** 管理员在籍贯或学校所在区域列填写一个合法的区域完整路径
- **THEN** 系统 MUST 将该值解析为对应的 `province_code`、`city_code`、`county_code`
- **AND** 系统 MUST 允许只保存省级，或省级加市级，或省级加市级加区县

#### Scenario: 非法区域路径被拦截
- **WHEN** 上传文件中的籍贯或学校所在区域不是合法的完整路径值，或层级组合不合法
- **THEN** 系统 MUST 拒绝导入该行
- **AND** 系统 MUST 返回明确的区域匹配或层级错误原因

#### Scenario: 非法学校、专业或学历选项被拦截
- **WHEN** 上传文件中的学校、专业或学历不能唯一匹配到当前主数据中的官方编码
- **THEN** 系统 MUST 拒绝导入该行
- **AND** 系统 MUST 返回明确的学校、专业或学历匹配错误原因

#### Scenario: 非法招考年份被拦截
- **WHEN** 上传文件中的招考年份不是合法的 4 位年份值
- **THEN** 系统 MUST 拒绝导入该行
- **AND** 系统 MUST 返回明确的招考年份格式错误原因

#### Scenario: 返回导入失败明细
- **WHEN** 上传文件中存在非法字典值、学校不存在、专业不存在、学历不存在、区域路径无法匹配、招考年份格式错误、联系方式错误或重复数据
- **THEN** 系统 MUST 返回逐行错误信息，至少包含失败行号和失败原因

#### Scenario: 导入失败返修文件
- **WHEN** 管理员重新上传一个仅比标准模板多出末尾 `失败原因` 列的失败数据返修文件
- **THEN** 系统 MUST 按前置业务列继续执行相同的逐行校验和导入逻辑
- **AND** 系统 MUST 忽略 `失败原因` 列内容，不因该列存在而拒绝导入整个文件
