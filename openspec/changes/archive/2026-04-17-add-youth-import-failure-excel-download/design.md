## Context

当前青年信息 Excel 导入在服务层逐行校验上传数据，并把失败结果汇总为“行号 + 原因”返回到页面。管理员可以看到失败明细，但仍需回到原始文件中手工筛选失败行、复制失败数据并重新整理成新的导入文件，返修成本较高。

现有系统已经具备两项与本次变更直接相关的基础能力：

- 青年信息标准导入模板由 Apache POI 生成，并包含隐藏工作表、命名区域和下拉选项来源。
- 青年信息导入会逐行解析 Excel，能够在失败时定位行号并生成统一错误消息。

本次设计需要在不改变现有青年导入主流程的前提下，把失败数据沉淀为一个可下载、可直接返修、并可再次上传的 Excel 文件，同时尽量复用现有模板生成逻辑，避免出现模板口径分叉。

## Goals / Non-Goals

**Goals:**
- 在青年导入存在失败数据时，为管理员提供失败数据 Excel 下载能力。
- 失败数据文件仅包含失败的原始数据行，保留原始填写值，不自动修正。
- 失败数据文件继续复用导入模板的字段顺序、下拉选项和隐藏数据工作表。
- 在失败数据文件末尾增加 `失败原因` 列，帮助管理员直接在 Excel 中修订。
- 允许管理员将带 `失败原因` 列的失败数据文件再次上传，系统导入时忽略该列。

**Non-Goals:**
- 不改变标准导入模板下载文件的字段结构。
- 不扩展企业、学校、专业等其他模块的导入失败返修文件能力。
- 不引入异步任务、对象存储或长期保存历史失败文件。
- 不升级为字段级错误定位或单元格批注能力。

## Decisions

### 1. 失败数据文件基于标准导入模板二次生成

失败数据 Excel 不从零开始手写工作簿，而是基于现有标准导入模板生成逻辑创建，再把失败数据行写回主工作表，最后追加 `失败原因` 列。

这样做的原因：
- 能继承现有隐藏 sheet、命名区域和下拉逻辑，避免维护两套 Excel 结构。
- 能确保失败数据文件与标准模板的受控字段口径保持一致。
- 失败数据文件和标准模板只在“是否含失败数据、是否多一列失败原因”上有差异，更利于后续维护。

**Alternatives considered**
- 单独实现一套失败文件工作簿生成逻辑：结构重复，容易与标准模板偏离。
- 直接导出纯数据表，不带下拉：管理员返修时体验较差。

### 2. 导入结果对象扩展为同时保存错误描述与失败行原始值

当前导入结果只记录失败行号和失败原因，无法直接重建失败 Excel。为支持失败文件生成，导入流程需要在失败时额外保留该行各列的原始字符串值，并与错误原因绑定。

建议把“失败行数据”作为导入结果中的一类显式结构保存，至少包括：
- 原始数据列值列表
- 失败原因
- 原始 Excel 行号

页面展示仍可继续使用行号和失败原因；失败 Excel 生成则消费失败行原始值。

**Alternatives considered**
- 失败后重新读取原始文件并按行号回捞数据：逻辑脆弱，增加二次解析成本。
- 仅保存错误原因，不保存原始值：无法生成失败返修文件。

### 3. 表头校验调整为兼容末尾 `失败原因` 列

现有导入严格要求表头和列数与标准模板完全一致。为了让失败数据文件能直接再次导入，表头校验需要兼容两种合法结构：

- 标准模板 11 列
- 标准模板 11 列 + 末尾 `失败原因` 列

解析业务字段时，系统仍只消费前 11 列，忽略最后的 `失败原因` 列内容。

**Alternatives considered**
- 要求管理员重新导入前手工删除 `失败原因` 列：增加返修摩擦，闭环不完整。
- 把失败原因写到批注或隐藏 sheet 中：列结构更干净，但返修可见性较差，且不符合当前业务偏好。

### 4. 下载入口保持二段式交互，不改现有导入 POST 响应形态

当前青年导入使用标准表单上传并在完成后重定向回列表页。失败文件下载不直接复用导入 POST 响应，而是在导入完成后通过页面上的下载入口单独获取失败 Excel。

推荐实现为：
- 导入完成后，失败数据暂存在服务端返回对象或短期页面上下文中。
- 页面在存在失败数据时展示“下载失败数据”按钮。
- 按钮访问新的失败文件下载接口并生成 Excel。

这样可以保持现有成功/失败提示、导入结果区域和页面跳转行为稳定。

**Alternatives considered**
- 让导入接口直接返回文件流：会破坏当前重定向和页面提示流程。
- 上传完成后自动触发浏览器下载：依赖前端重构为 Ajax 上传，改动面更大。

## Risks / Trade-offs

- [失败数据暂存依赖当前请求结果上下文] → 通过明确失败下载入口只服务本次导入结果，避免引入长期持久化复杂度。
- [失败文件比标准模板多一列，模板逻辑出现分叉] → 将差异收敛为“标准模板 + 末尾错误列”，复用同一套字段和下拉生成逻辑。
- [错误原因为行级文本，未精确定位到具体字段] → 先满足快速返修需求，后续若反馈仍不够清晰，再评估字段级标注。
- [失败数据量大时 Excel 生成成本增加] → 失败文件只包含失败行，通常规模可控，继续使用现有 POI 方案。

## Migration Plan

1. 扩展青年导入结果结构，支持保存失败行原始值和错误原因。
2. 新增失败数据 Excel 生成逻辑，基于标准模板回填失败行并追加 `失败原因` 列。
3. 调整导入表头校验和解析逻辑，兼容末尾 `失败原因` 列。
4. 在青年导入结果区域增加失败文件下载入口。
5. 补充服务层与集成测试，覆盖失败下载和失败文件再次导入场景。

## Open Questions

- 失败数据下载入口是只在本次重定向后的页面提供，还是需要支持刷新后再次下载；当前方案默认仅覆盖本次导入后的即时下载场景。
