## ADDED Requirements

### Requirement: 管理员可按在校学生发起招聘匹配
系统 MUST 允许管理员在在校学生详情抽屉中基于单条学生记录发起招聘匹配，并展示与该学生条件对应的招聘岗位结果列表。

#### Scenario: 从学生详情进入匹配岗位结果
- **WHEN** 管理员在一名在校学生的详情抽屉中点击“匹配招聘信息”
- **THEN** 系统 MUST 打开该学生对应的招聘匹配结果抽屉
- **AND** 系统 MUST 展示学生姓名、已生效的匹配条件和匹配到的招聘岗位数量

#### Scenario: 匹配结果仅返回招聘岗位
- **WHEN** 系统根据学生条件查询匹配结果
- **THEN** 系统 MUST 仅返回未删除的招聘岗位记录
- **AND** 系统 MUST 不返回学生记录或其他业务实体作为匹配结果

### Requirement: 学生匹配岗位条件按显式规则生效
系统 MUST 仅对学生已具备的专业、学历、学校类别、学校标签四类条件执行岗位筛选；学生侧缺失的维度 MUST 被自动忽略。不同维度之间 MUST 按 AND 组合，同一维度内的多选值 MUST 按 OR 命中，并且匹配规则 MUST 与岗位匹配学生保持完全一致。

#### Scenario: 学生专业匹配岗位专业要求
- **WHEN** 当前学生保存了专业官方编码，且招聘岗位配置了一个或多个专业要求
- **THEN** 系统 MUST 仅返回专业要求集合中包含该学生专业编码的岗位
- **AND** 系统 MUST 不得仅按专业类别或自由文本描述替代专业匹配

#### Scenario: 学生学历匹配岗位学历要求
- **WHEN** 当前学生保存了学历编码，且招聘岗位配置了一个或多个学历要求
- **THEN** 系统 MUST 仅返回学历要求集合中包含该学生学历编码的岗位
- **AND** 系统 MUST 不得自动放宽为更高或更低学历均可命中

#### Scenario: 学生学校类别匹配岗位学校类别要求
- **WHEN** 当前学生关联学校主数据存在学校类别，且招聘岗位配置了一个或多个学校类别要求
- **THEN** 系统 MUST 基于学生学校关联的学校类别 ID 进行筛选
- **AND** 学生命中其中任一已选学校类别即可通过该维度

#### Scenario: 学生学校标签匹配岗位学校标签要求
- **WHEN** 当前学生关联学校主数据存在一个或多个学校标签，且招聘岗位配置了一个或多个学校标签要求
- **THEN** 系统 MUST 基于学生学校关联的标签集合与岗位标签集合是否存在交集进行筛选
- **AND** 学生命中其中任一已选学校标签即可通过该维度

#### Scenario: 学生缺失维度时忽略对应岗位条件
- **WHEN** 当前学生缺失专业、学历、学校类别或学校标签中的任一维度
- **THEN** 系统 MUST 在匹配查询中忽略该维度
- **AND** 系统 MUST 不因为该维度缺失而直接判定所有岗位都不匹配
