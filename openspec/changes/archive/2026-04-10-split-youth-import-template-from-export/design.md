## Context

当前青年信息 Excel 模板下载、Excel 导入和 Excel 导出共用同一套表头定义，代码中由 `YouthInfoServiceImpl` 的统一表头数组驱动模板生成、导入表头校验和导出列顺序。这种实现简单，但会把“采集模板”和“业务导出”强绑定在一起，一旦希望缩减导入模板字段，就会连带压缩导出内容。

本次需求不是修改青年信息数据结构，而是重新定义 Excel 交互边界：导入模板只保留基础采集字段，导出继续保留完整业务字段。因此需要将当前“一套表头多处复用”的实现拆成“导入模板/导入校验一套表头，导出一套表头”。

## Goals / Non-Goals

**Goals:**
- 将青年信息 Excel 导入模板字段调整为精简版 11 列。
- 让 Excel 导入严格按照新的精简表头和顺序进行校验与解析。
- 保持 Excel 导出字段不变，继续输出完整业务字段。
- 在规格层明确模板/导入字段与导出字段允许不同。

**Non-Goals:**
- 不修改 `youth_info` 表结构或青年信息表单字段。
- 不修改青年信息页面的新增、编辑、详情展示逻辑。
- 不新增新的导出字段，也不删减当前导出字段。

## Decisions

### 1. 将导入模板表头与导出表头拆分为两套定义

实现上不再让模板、导入、导出共用同一个表头常量，而是分别维护：
- 导入模板/导入校验表头
- 导出表头

原因：
- 这是满足“模板精简、导出保持完整”最直接且最清晰的实现方式。
- 可以避免后续继续调整导入模板时意外影响导出。

备选方案：
- 仍然共用一套表头，并在导出时动态补列：逻辑更绕，测试复杂度更高。

### 2. 新导入模板只保留基础采集字段

导入模板字段调整为：
`姓名、性别、民族、出生年月、籍贯、招考年份、学历、学校、学校所在区域、专业、联系方式`

移除：
`政治面貌、学位、毕业时间、就业方向`

原因：
- 这些字段不是当前最小采集闭环的必要项。
- 模板越短，基层采集成本越低，错误率也更可控。

### 3. 导入解析对移除字段改为不依赖模板提供

导入逻辑需要随模板变化同步调整：
- 不再从 Excel 中读取政治面貌、学位、毕业时间、就业方向
- 保留原有受控字段校验：性别、民族、学历、学校、学校所在区域、专业、招考年份
- 其他未出现在模板中的业务字段由系统保持为空或沿用现有默认行为

原因：
- 导入模板已经不承载这些字段，导入逻辑不能继续把它们当成必填列。

### 4. 导出继续维持完整业务口径

导出逻辑保持现有完整字段集合，不因模板精简而删列。

原因：
- 导出面向业务查看和数据回传，信息完整性优先。
- 这也是本次需求明确要求保持不变的部分。

## Risks / Trade-offs

- [模板与导出字段不同可能让用户困惑] → 在 spec 和实现中明确“模板用于采集，导出用于业务查看”，必要时后续再补页面说明文案。
- [导入解析列号调整容易出错] → 同步更新模板测试、导入测试和导出测试，分别校验三条链路。
- [被移除字段改为非模板输入后，导入数据完整度下降] → 保留页面维护和导出完整字段，不影响后续人工补录和业务查看。
