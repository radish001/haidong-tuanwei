## MODIFIED Requirements

### Requirement: 后台统一视觉壳子
系统必须为后台管理端提供统一的驾驶舱风格页面壳子，覆盖侧边导航、顶部标题区、内容面板和全局主题配色，使管理员在不同模块间切换时保持一致的视觉体验；该页面壳子 MUST 在共享顶栏承担主标题表达的前提下，减少页面内部重复的大标题卡、说明卡和嵌套容器，保持更紧凑、规整的工作区层级。

#### Scenario: 打开任一后台业务页面
- **WHEN** 已登录管理员进入首页、青年信息、企业信息、招聘信息、政策管理或分析页面
- **THEN** 系统展示统一的深色驾驶舱风格页面壳子，包括左侧导航、顶部标题区、统一面板边框和一致的页面背景层次
- **AND** 页面内部 MUST 避免重复表达同一页面标题或说明信息，优先将主要注意力留给当前工作区内容

### Requirement: 后台通用组件风格一致
系统必须对按钮、筛选栏、输入框、表格、提示信息和操作区采用统一的视觉规范，避免不同业务页面之间出现明显割裂的样式表现；这些通用组件 MUST 以更紧凑的间距、稳定的对齐关系和一致的密度呈现，减少不必要的大圆角、大阴影、大留白和重复卡片层级。

#### Scenario: 查看不同模块的列表和表单页面
- **WHEN** 管理员在不同业务模块之间切换并查看列表页、表单页或详情页
- **THEN** 系统以统一的组件风格呈现筛选区、操作按钮、表格、表单控件和提示信息，并保持一致的间距、描边、高亮和交互反馈
- **AND** 列表页中的筛选、操作、表格和分页 MUST 形成紧凑、整齐的工作台结构，而不是被分散在多个装饰性区块中

## ADDED Requirements

### Requirement: 后台看板和分析页采用规整网格
系统 MUST 将首页和数据分析页组织为规整、可响应的看板网格，使地图、统计数字、图表和摘要列表在大屏和常规桌面下都保持明确对齐关系；页面不得依赖多层嵌套卡片和重复标题制造视觉层次。

#### Scenario: 打开首页或分析页
- **WHEN** 管理员进入首页或任一数据分析页面
- **THEN** 系统 MUST 通过规则网格展示地图、统计卡、图表和摘要数据
- **AND** 页面中的主要视觉焦点 MUST 是地图、数字和图表本身，而不是额外的装饰性标题卡或说明卡

### Requirement: 后台抽屉呈现为工作侧板
系统 MUST 将右侧抽屉页面呈现为统一的工作侧板，确保头部、正文和底部操作区之间的层级关系明确，并在桌面端和移动端都保持稳定的可用性；抽屉不得再次模拟一个完整页面壳子。

#### Scenario: 在列表页中打开新增、编辑或详情抽屉
- **WHEN** 管理员从任一后台列表页打开新增、编辑或详情内容
- **THEN** 系统 MUST 以紧凑、规整的侧板结构展示标题、表单或详情信息以及底部操作区
- **AND** 抽屉内部 MUST 避免重复的页面级标题壳子、过大的留白和不整齐的表单排布
