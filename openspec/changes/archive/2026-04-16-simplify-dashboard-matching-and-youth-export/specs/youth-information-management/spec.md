## MODIFIED Requirements

### Requirement: 青年信息查询与维护
系统 MUST 支持通过受控字段和区域条件查询青年信息，并在当前可见的在校大学生列表页提供与后台统一规范一致的分页、局部刷新、抽屉维护和删除确认交互；列表页与表单页 MUST 采用正式、精简、适合业务管理场景的展示文案。学校、专业、学历和学位相关查询与维护 MUST 基于主数据官方编码完成，但页面展示仍 MUST 提供业务可读的学校名称、专业名称、学历名称和学位名称；专业类别相关筛选、存储和展示 MUST 统一以字典值作为保存与查询口径，并以字典标签作为页面主要展示内容。系统同时 MUST 支持按招聘岗位派生出的专业、学历和学校类别条件筛选在校大学生，并支持在校学生详情中按完全一致的专业、学历和学校类别条件反向筛选招聘岗位。当前在校大学生列表 MUST 在“出生年月”后展示“民族”字段，并不得继续以“就业方向”作为该列表的固定展示列。当前青年信息维护表单中的籍贯、学校所在地和现居住地控件 MUST 与其他常规表单字段保持一致的列宽规则，不得长期以明显更长的整行控件破坏表单对齐。当前在校大学生列表页中的专业筛选 MUST 支持由专业类别驱动的级联选择。

#### Scenario: 按条件筛选青年信息
- **WHEN** 管理员使用籍贯、性别、民族、政治面貌、学历、学位、学校、专业名称、姓名等条件查询青年信息
- **THEN** 系统仅返回符合条件的青年记录
- **AND** 系统 MUST 允许管理员通过受控学校、专业、学历和学位选项发起查询，即使底层数据按编码存储
- **AND** 系统以统一视觉风格刷新当前在校大学生列表

#### Scenario: 专业类别驱动专业级联筛选
- **WHEN** 管理员在青年信息列表页先选择一个专业类别
- **THEN** 系统 MUST 将专业下拉选项限制为该专业类别下的专业
- **AND** 当专业类别为空时系统 MUST 允许专业下拉恢复为全部专业选项

#### Scenario: 青年列表使用统一管理布局
- **WHEN** 管理员进入青年信息列表页
- **THEN** 系统 MUST 在页面上方展示查询筛选区
- **AND** 系统 MUST 在查询区域下方右侧展示新增、删除等操作按钮
- **AND** 系统 MUST 在列表底部展示分页信息、页号选择和每页条数选择

#### Scenario: 青年列表展示民族字段
- **WHEN** 管理员查看在校大学生列表中的基础信息列
- **THEN** 系统 MUST 在“出生年月”后展示“民族”列
- **AND** 系统不得继续在该列表中以“就业方向”作为固定展示列

#### Scenario: 青年列表分页局部刷新
- **WHEN** 管理员执行查询、切换页号或调整每页条数
- **THEN** 系统 MUST 仅刷新青年信息列表区域及其分页区域
- **AND** 系统不得触发整页跳转刷新

#### Scenario: 青年详情与编辑通过右侧抽屉打开
- **WHEN** 管理员在青年信息列表中点击新增、详情或编辑
- **THEN** 系统 MUST 在当前页面右侧滑出抽屉面板承载对应内容
- **AND** 系统不得通过整页跳转进入独立详情页或编辑页

#### Scenario: 编辑已存在的青年信息
- **WHEN** 管理员在右侧抽屉中修改一条已保存的青年信息并提交合法内容
- **THEN** 系统保存修改结果
- **AND** 系统关闭抽屉并刷新当前青年列表数据

#### Scenario: 删除青年信息前要求确认
- **WHEN** 管理员删除一条青年信息或批量删除多条青年信息
- **THEN** 系统 MUST 先展示平台风格的确认模态框
- **AND** 仅当管理员确认后系统才执行删除

#### Scenario: 按岗位匹配条件筛选在校大学生
- **WHEN** 系统根据某条招聘岗位已填写的专业要求、学历要求和学校类别查询匹配学生
- **THEN** 系统 MUST 仅返回符合所有已填写维度条件的在校大学生
- **AND** 系统 MUST 忽略该岗位中未填写的匹配维度
- **AND** 系统 MUST 不得将学校标签作为学生匹配条件
- **AND** 系统 MUST 在结果中回显可读的学校名称、专业名称、学历名称和专业类别

#### Scenario: 从学生详情抽屉进入匹配招聘结果
- **WHEN** 管理员在一名在校学生的详情抽屉中点击“匹配招聘信息”
- **THEN** 系统 MUST 在当前页面右侧打开招聘匹配结果抽屉
- **AND** 该结果抽屉 MUST 覆盖当前学生详情抽屉，而不是直接关闭详情后跳回列表

#### Scenario: 关闭匹配招聘结果后返回学生详情抽屉
- **WHEN** 管理员关闭由学生详情进入的招聘匹配结果抽屉
- **THEN** 系统 MUST 恢复原学生详情抽屉内容
- **AND** 系统 MUST 不得直接返回青年信息列表页

## ADDED Requirements

### Requirement: 学生匹配招聘结果不展示学校标签
系统 MUST 在学生详情进入的招聘匹配结果页面中去除学校标签条件摘要，避免将学校标签误解为当前仍生效的匹配维度。

#### Scenario: 查看学生匹配招聘结果摘要
- **WHEN** 管理员在学生详情抽屉中打开“匹配招聘信息”结果
- **THEN** 系统 MUST 仅回显当前参与匹配的专业、学历和学校类别等条件摘要
- **AND** 系统 MUST 不展示学校标签摘要占位或空值提示

### Requirement: 学生信息导出顺序与导入模板一致
系统 MUST 让学生信息导出文件使用与导入模板完全一致的字段集合与字段顺序，便于业务人员直接基于导出结果整理后再次导入。

#### Scenario: 导出学生信息
- **WHEN** 管理员导出学生信息 Excel
- **THEN** 系统 MUST 按“姓名、性别、民族、出生年月、籍贯、招考年份、学历、学校、学校所在区域、专业、联系方式”的顺序输出列
- **AND** 系统 MUST 不输出导入模板中不存在的额外列
