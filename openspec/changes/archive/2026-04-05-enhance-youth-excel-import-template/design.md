## Context

当前青年信息 Excel 模板已经具备基础表头、隐藏数据源 sheet 和部分下拉校验，但模板中区域字段仍使用平铺区域名称列表，导入也仅按单个区域名称匹配，无法与页面上“省 / 省市 / 省市区县均可”的三级区域规则保持一致。与此同时，学校、专业、学历等字段虽然已有主数据或字典来源，但模板与导入规则还缺少一份明确、统一的设计约束，尤其需要明确区域字段也必须复用区域主数据而不是自由拼写。

本次改动只针对青年信息导入导出，不扩展到企业、招聘或新的 Excel 交互方式。用户已明确接受继续使用长下拉硬选方案，并以 WPS 为主要使用环境，因此不引入宏、脚本或复杂的 Excel 搜索控件。

## Goals / Non-Goals

**Goals:**
- 保持青年信息导入模板为固定 13 列，不增加区域拆分列。
- 让模板中的受控字段全部直接复用现有字典、学校主数据、专业主数据和区域主数据。
- 将区域字段统一为完整路径单列下拉值，并支持省、省 / 市、省 / 市 / 区县三种合法层级。
- 让导入解析按完整路径解析区域并写回三级区域代码，避免仅按末级名称匹配造成歧义。
- 保持导出列与模板列对齐，便于“导出后再导入”。

**Non-Goals:**
- 不为 Excel 模板实现网页级联式地区选择体验。
- 不为 Excel 下拉引入宏、ActiveX、VBA 或其他依赖客户端能力的搜索功能。
- 不调整青年信息页面表单或列表页的现有字段结构。
- 不扩展企业、招聘模块的 Excel 导入导出。

## Decisions

### 1. 模板字段继续保持 13 列单列表头

保持现有模板列结构不变，继续使用：
姓名、性别、出生年月、民族、政治面貌、籍贯、学历、学校、学校所在区域、专业、毕业时间、就业方向、联系方式。

这样可以避免额外的列结构迁移，也与现有导出结构保持兼容。

**Alternatives considered**
- 将区域拆成省/市/区三列：导入更容易实现，但会破坏用户已确认的 13 列模板结构。
- 将学校、专业等字段改为编码列：便于导入，但不利于人工填写与核对。

### 2. 所有受控字段均直接复用现有主数据或字典值

模板中的性别、民族、政治面貌、学历字段来自 `sys_dict_item`；学校来自学校主数据；专业来自专业主数据；籍贯和学校所在区域来自区域主数据。模板展示值和导入校验值保持同一口径，避免“模板选项”和“系统允许值”不一致。

学历范围不在模板里单独写死，而是完全以字典管理中的当前配置为准。

**Alternatives considered**
- 在模板中单独维护一份静态学历选项：会与字典管理脱节，后续维护成本更高。

### 3. 区域字段采用完整路径单列下拉

`籍贯` 和 `学校所在区域` 两列的下拉项统一基于区域主数据生成完整路径，支持三类合法值：
- `省`
- `省 / 市`
- `省 / 市 / 区县`

模板中的区域选项列表将同时包含这三类路径值。例如：
- `青海省`
- `青海省 / 海东市`
- `青海省 / 海东市 / 乐都区`

这既保留单列录入形式，也与页面允许“部分层级选择”的规则一致。

**Alternatives considered**
- 仅提供区县级完整路径：无法满足只填省或省市的业务规则。
- 仅提供末级区域名称：在重名地区场景下有歧义，导入不稳定。

### 4. 导入时按完整路径解析区域，不再按单个区域名称匹配

区域解析逻辑将从“名称 -> 区域对象”的单值匹配，升级为“完整路径 -> 区域层级选择”的解析。解析结果最终仍写入 `province_code`、`city_code`、`county_code` 三段字段，并复用现有 `RegionSelectionSupport` 做层级合法性校验。

这意味着：
- `青海省` 写入 province code
- `青海省 / 海东市` 写入 province + city code
- `青海省 / 海东市 / 乐都区` 写入 province + city + county code

**Alternatives considered**
- 导入时拆分末级名称再反查父级：对重名区域不可靠，且规则不够显式。

### 5. 导出结果与模板字段口径保持一致

导出列继续使用 13 列顺序，并对区域列导出完整路径值，确保导出文件可以直接作为人工修订后重新导入的基础。

**Alternatives considered**
- 导出仍保留现有显示口径而模板使用新口径：会导致“导出后不可直接回填导入”。

## Risks / Trade-offs

- [区域下拉项会更长] → 通过 WPS 原生下拉输入筛选能力缓解，且用户已接受长下拉硬选方案。
- [区域、学校、专业选项量大，模板生成时间可能变长] → 继续使用隐藏 sheet 与命名区域，避免引入更复杂的 Excel 构造逻辑。
- [Excel 与 WPS 下拉交互体验存在差异] → 以 WPS 为主要使用环境设计，同时确保在原版 Excel 中仍能通过标准下拉选择合法值。
- [导出改为完整路径后与旧模板视觉不完全一致] → 用“导出可回填导入”的一致性收益覆盖该变化。

## Migration Plan

1. 调整模板下拉生成逻辑，新增区域完整路径选项集。
2. 调整导入解析逻辑，支持完整路径到三级区域代码的转换。
3. 调整导出逻辑，使区域列输出完整路径。
4. 使用新模板验证以下场景：
   - 省级区域导入
   - 省市级区域导入
   - 省市区县级区域导入
   - 字典项、学校、专业非法值拦截
   - 导出结果可作为再次导入的合法样例

## Open Questions

- 当前不追加参考 sheet 搜索说明页，默认依赖 WPS 原生下拉输入筛选；若后续用户反馈 Excel 兼容性问题，再单独评估是否补充参考页。
