## MODIFIED Requirements

### Requirement: 青年信息导入模板下载
系统 MUST 提供青年信息 Excel 导入模板下载能力，模板包含一个可填写主工作表和多个用于下拉选项的数据隐藏工作表，并与当前青年信息受控字段、学校主数据、专业主数据和区域主数据口径保持一致。

#### Scenario: 下载青年信息导入模板
- **WHEN** 管理员在青年信息模块中点击下载导入模板
- **THEN** 系统下载一个包含已确认 13 列表头和隐藏数据工作表的 Excel 模板文件

#### Scenario: 模板包含受控下拉选项
- **WHEN** 管理员打开下载后的模板文件
- **THEN** 性别、民族、政治面貌、学历字段 MUST 使用字典数据作为下拉来源
- **AND** 学校、专业字段 MUST 使用对应主数据作为下拉来源
- **AND** 籍贯、学校所在区域字段 MUST 使用区域主数据作为下拉来源

#### Scenario: 区域字段使用完整路径下拉
- **WHEN** 管理员查看模板中的籍贯或学校所在区域列
- **THEN** 系统 MUST 提供完整路径形式的区域下拉值
- **AND** 区域下拉值 MUST 同时支持 `省`、`省 / 市`、`省 / 市 / 区县` 三种合法层级

### Requirement: 青年信息 Excel 导入校验
系统 MUST 在保存前逐行校验上传的青年信息 Excel 数据，并按照当前字典、学校、专业和区域主数据规则拦截非法值或重复数据。

#### Scenario: 导入合法青年数据
- **WHEN** 管理员上传按模板正确填写且数据合法的 Excel 文件
- **THEN** 系统将这些记录保存到统一青年信息表中，并写入当前分类对应的 `youth_type`

#### Scenario: 区域字段按完整路径导入
- **WHEN** 管理员在籍贯或学校所在区域列填写一个合法的区域完整路径
- **THEN** 系统 MUST 将该值解析为对应的 `province_code`、`city_code`、`county_code`
- **AND** 系统 MUST 允许只保存省级，或省级加市级，或省级加市级加区县

#### Scenario: 非法区域路径被拦截
- **WHEN** 上传文件中的籍贯或学校所在区域不是合法的完整路径值，或层级组合不合法
- **THEN** 系统 MUST 拒绝导入该行
- **AND** 系统 MUST 返回明确的区域匹配或层级错误原因

#### Scenario: 返回导入失败明细
- **WHEN** 上传文件中存在非法字典值、学校不存在、专业不存在、区域路径无法匹配、日期格式错误、联系方式错误或重复数据
- **THEN** 系统 MUST 返回逐行错误信息，至少包含失败行号和失败原因

### Requirement: 青年信息导出
系统 MUST 允许管理员在青年信息分类页面中按当前筛选范围导出数据，并使导出列结构与导入模板保持一致。

#### Scenario: 导出当前筛选结果
- **WHEN** 管理员在青年分类页面中设置筛选条件后执行导出
- **THEN** 系统下载仅包含当前分类和当前筛选结果的 Excel 文件

#### Scenario: 导出列结构与模板一致
- **WHEN** 系统生成青年信息导出文件
- **THEN** 导出文件 MUST 使用与导入模板一致的 13 列字段顺序
- **AND** 区域字段 MUST 使用完整路径值输出
