## Context

当前后台列表筛选区混用了两类单选控件：普通字段通过 `bootstrap-enhancements.js` 增强为 Tom Select，区域字段通过 `region-step-select.js` 使用 treeselectjs。两类控件在初次加载时勉强保持相近外观，但在空态表达、AJAX 局部刷新后的重建逻辑，以及下拉面板尺寸策略上并不一致，导致同一筛选栏里出现颜色、回显和可读性不统一的问题。

本次修复属于共享查询组件行为调整，会影响所有复用 `data-ajax-list-root`、`data-ajax-form` 和 `[data-region-step]` 的后台列表页，因此需要在共享初始化和共享样式层处理，而不是只修补青年页面模板。

## Goals / Non-Goals

**Goals:**
- 保证后台列表筛选控件在“未选择”“全部”“已选择具体值”三种状态下具有一致、可读的视觉表达。
- 保证局部刷新后普通筛选下拉和区域筛选都能恢复正确的增强行为。
- 保证区域下拉面板能够完整展示较长的区域路径，不受输入框宽度限制。

**Non-Goals:**
- 不处理青年列表字段顺序调整。
- 不处理新增表单中区域输入框过长的问题。
- 不在本次变更中处理数据库索引优化。

## Decisions

### 1. 保留现有两套组件，但统一空态语义

继续使用 Tom Select 处理普通单选、Treeselect 处理区域级联，避免扩大组件替换范围。空态统一规则为：未选中时展示占位提示，不把占位文字写成真实值；只有用户明确选择具体项后，才展示真实选中内容。

备选方案是将区域控件也替换成 Tom Select 体系，但这会引入级联能力重做和更大范围模板调整，不适合本次快速修复。

### 2. 在列表局部刷新后统一重建增强控件

`list-pagination.js` 在替换列表 fragment 后，需要重新执行普通下拉增强和区域控件初始化，确保首次加载和 AJAX 刷新后的 DOM 状态一致。这样可以解决空值回显退回原生 select、显示成普通白字的问题。

备选方案是让模板直接输出完全不同的空选项文本以兼容原生 select，但这只能缓解表现问题，不能解决增强逻辑缺失。

### 3. 区域下拉面板独立于输入框宽度

区域下拉面板继续挂载在当前控件容器下，但样式上给予独立的最小宽度和更高层级，使其可以在狭窄筛选格子里展开更宽的可读面板，同时避免被相邻内容压住。

备选方案是将下拉面板直接挂到 `body`，但会引入额外的定位同步和滚动跟随复杂度，本次不采用。

## Risks / Trade-offs

- [共享脚本改动影响多个列表页] → 通过保持初始化入口不变、仅补齐刷新后的增强步骤来降低回归范围。
- [区域控件空态逻辑调整可能影响已选值回显] → 保留 hidden input 作为真实查询值来源，只调整展示层输入框的空态表达。
- [更宽的下拉层可能在小屏上压住周边元素] → 使用最小宽度和层级提升，而不是无限扩张面板宽度。
