## ADDED Requirements

### Requirement: 青年信息支持人工排序字段
系统 MUST 在青年信息维护链路中提供一个名为“排序”的可空整数型字段。管理员 MUST 在新增与编辑青年信息时填写或清空该字段，系统 MUST 在青年信息详情中展示当前排序值，并保持详情页只读不直接编辑该字段。

#### Scenario: 在新增或编辑时维护排序值
- **WHEN** 管理员在青年信息新增或编辑表单中填写一个整数排序值或留空提交
- **THEN** 系统 MUST 保存该排序值或空值
- **AND** 系统 MUST 允许该字段不填

#### Scenario: 在详情中查看排序值
- **WHEN** 管理员打开一条青年信息详情
- **THEN** 系统 MUST 展示该记录当前保存的“排序”值
- **AND** 系统 MUST 不在详情页提供直接编辑该字段的交互

### Requirement: 青年信息列表按人工排序优先展示
系统 MUST 在青年信息列表中优先按人工排序字段展示记录。已填写“排序”的记录 MUST 排在未填写记录之前，并按排序值升序排列；当多个记录的排序值相同时，系统 MUST 继续按 `id desc` 排序；未填写排序值的记录 MUST 继续按现有 `id desc` 规则排序。青年信息列表页 MUST 不再显示“序号”列。

#### Scenario: 已填写排序值的记录优先显示
- **WHEN** 青年信息列表同时存在已填写排序值和未填写排序值的记录
- **THEN** 系统 MUST 先展示所有已填写排序值的记录
- **AND** 这些记录 MUST 按排序值从小到大排列

#### Scenario: 排序值相同或为空时继续使用默认顺序
- **WHEN** 多条青年信息的排序值相同，或这些记录都未填写排序值
- **THEN** 系统 MUST 在对应分组内继续按 `id desc` 排列

#### Scenario: 列表页不再显示序号列
- **WHEN** 管理员查看青年信息列表
- **THEN** 系统 MUST 不再渲染“序号”列
- **AND** 系统 MUST 保留现有分页信息与其他业务列
