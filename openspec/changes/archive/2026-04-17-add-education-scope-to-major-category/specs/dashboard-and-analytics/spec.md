## MODIFIED Requirements

### Requirement: 分类数据分析展示
系统 MUST 保留按青年分类提供统计分析能力，但当前后台页面层仅对管理员展示在校大学生分析视图。在校大学生分析页 MUST 将原"专业类别分布"拆分为"本科专业类别分布"和"专科专业类别分布"两张独立图表，分别展示不同学历层次的专业类别分布。

#### Scenario: 在校大学生分析页展示本科专业类别分布
- **WHEN** 系统生成本科专业类别分布图
- **THEN** 系统 MUST 使用标题"本科专业类别分布"
- **AND** 系统 MUST 基于在校大学生记录中的专业官方编码关联专业主数据
- **AND** 系统 MUST 只统计所属学历层次包含"本科专业"的专业类别
- **AND** 系统 MUST 只统计学历字段值为"本科"的学生记录
- **AND** 系统 MUST 以竖向柱状图展示各专业类别的人数和占比

#### Scenario: 在校大学生分析页展示专科专业类别分布
- **WHEN** 系统生成专科专业类别分布图
- **THEN** 系统 MUST 使用标题"专科专业类别分布"
- **AND** 系统 MUST 位于本科专业类别分布图下方
- **AND** 系统 MUST 基于在校大学生记录中的专业官方编码关联专业主数据
- **AND** 系统 MUST 只统计所属学历层次包含"专科专业"的专业类别
- **AND** 系统 MUST 只统计学历字段值为"专科"的学生记录
- **AND** 系统 MUST 以竖向柱状图展示各专业类别的人数和占比

#### Scenario: 专业类别分布图按所属学历层次过滤
- **WHEN** 系统统计专业类别分布数据
- **THEN** 系统 MUST 同时检查专业类别的所属学历层次配置和学生的学历字段值
- **AND** 本科专业类别图 MUST 只包含 education_scopes 包含 UNDERGRADUATE 且学生 education_level 为本科的数据
- **AND** 专科专业类别图 MUST 只包含 education_scopes 包含 JUNIOR_COLLEGE 且学生 education_level 为专科的数据

#### Scenario: 无专科专业主数据时的图表展示
- **WHEN** 系统生成专科专业类别分布图但专业主数据中缺少专科专业记录
- **THEN** 系统 MUST 以空图表或"暂无数据"状态展示
- **AND** 系统 MUST 不报错或崩溃
