## ADDED Requirements

### Requirement: 权限配置数据使用物理删除
系统 MUST 将后台管理员、角色、菜单以及其授权关系视为当前权限配置，而不是需要保留软删除历史的业务内容。对于允许删除的权限配置记录，系统 MUST 执行物理删除，并同步清理相关关系数据。

#### Scenario: 删除角色或菜单配置
- **WHEN** 管理员删除一个允许移除的角色或菜单配置
- **THEN** 系统 MUST 物理删除对应记录
- **AND** 系统 MUST 同步清理其关联的授权关系，避免残留失效引用

#### Scenario: 删除用户配置
- **WHEN** 管理员删除一个允许移除的后台用户
- **THEN** 系统 MUST 物理删除该用户记录
- **AND** 系统 MUST 同步清理与该用户相关的角色关系记录

### Requirement: 权限关系不保留软删除历史
系统 MUST 将用户角色关系和角色菜单关系视为可重建的权限配置关系，并使用物理删除语义处理删除和重建操作。

#### Scenario: 重建用户角色关系
- **WHEN** 系统调整某个后台用户的角色分配
- **THEN** 系统 MUST 通过物理删除旧关系并写入新关系完成变更
- **AND** 系统 MUST 不因历史关系记录阻塞同一角色关系的重新创建

#### Scenario: 重建角色菜单关系
- **WHEN** 系统调整某个角色的菜单授权集合
- **THEN** 系统 MUST 通过物理删除旧关系并写入新关系完成变更
- **AND** 系统 MUST 不因历史关系记录阻塞同一菜单授权的重新创建
