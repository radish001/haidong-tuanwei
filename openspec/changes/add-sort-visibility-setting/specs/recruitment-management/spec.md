## MODIFIED Requirements

### Requirement: 招聘信息支持人工排序字段
系统 MUST 在招聘岗位维护链路中提供一个名为“排序”的可空整数型字段，并由系统设置中的全局显示开关统一控制该字段是否在页面层展示。当开关开启时，管理员 MUST 在新增与编辑招聘岗位时填写或清空该字段，系统 MUST 在招聘岗位详情中展示当前排序值，并保持详情页只读不直接编辑该字段。当开关关闭时，招聘岗位详情页与新增/编辑页 MUST 不再展示“排序”字段，但系统 MUST 继续保留记录中已存在的排序值，且不得因此改变列表排序规则。

#### Scenario: 开启显示后在新增或编辑时维护排序值
- **WHEN** 系统设置中的“是否显示排序设置”开关处于开启状态，且管理员在招聘岗位新增或编辑表单中填写一个整数排序值或留空提交
- **THEN** 系统 MUST 保存该排序值或空值
- **AND** 系统 MUST 允许该字段不填

#### Scenario: 开启显示后在详情中查看排序值
- **WHEN** 系统设置中的“是否显示排序设置”开关处于开启状态，且管理员打开一条招聘岗位详情
- **THEN** 系统 MUST 展示该记录当前保存的“排序”值
- **AND** 系统 MUST 不在详情页提供直接编辑该字段的交互

#### Scenario: 关闭显示后隐藏排序字段
- **WHEN** 系统设置中的“是否显示排序设置”开关处于关闭状态，且管理员打开招聘岗位详情页或新增/编辑页
- **THEN** 系统 MUST 不展示“排序”字段输入框或只读展示项
- **AND** 系统 MUST 不要求管理员在当前页面维护排序值

#### Scenario: 关闭显示后编辑记录保留已有排序值
- **WHEN** 系统设置中的“是否显示排序设置”开关处于关闭状态，且管理员编辑一条已存在排序值的招聘岗位并提交其他字段修改
- **THEN** 系统 MUST 保留该记录原有的排序值
- **AND** 系统 MUST 不因页面未展示该字段而将其回写为空值
