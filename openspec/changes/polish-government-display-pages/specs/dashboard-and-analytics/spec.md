## MODIFIED Requirements

### Requirement: 分类数据分析展示
系统 MUST 保留按青年分类提供统计分析能力，但当前后台页面层仅对管理员展示在校大学生分析视图。现有其他青年分类分析能力可以在后端继续保留，但页面不得展示毕业未就业、农村社区待业、创业青年等分类分析入口、页签或提示文案。当前可见的在校大学生分析页 MUST 继续展示基于业务数据实时聚合的基础画像图表与海东籍顶尖高校专题图表。所有涉及学校和专业主数据的在校大学生统计 MUST 基于青年信息中保存的学校官方编码和专业官方编码关联主数据，不得依赖学校名称或专业名称字符串匹配。当前可见分析页 MUST 以正式、精简、适合政府展示的页面结构和文案呈现统计结果。

#### Scenario: 打开数据分析页面
- **WHEN** 管理员进入数据分析模块
- **THEN** 系统 MUST 默认展示在校大学生分析页面
- **AND** 页面层不得展示其他青年分类分析的页签、子菜单或并列入口

#### Scenario: 后端仍保留其他分类分析能力
- **WHEN** 系统后端仍保留毕业未就业、农村社区待业或创业青年相关分析路由、模型或聚合逻辑
- **THEN** 本次页面优化 MUST 不要求删除这些后端能力
- **AND** 当前后台页面仍 MUST 仅暴露在校大学生分析视图

#### Scenario: 在校大学生分析页展示固定基础画像图
- **WHEN** 管理员进入在校大学生分析页
- **THEN** 系统 MUST 展示院校层次分布、专业类别分布、民族分类占比、学历分布、性别分布五类固定图表

#### Scenario: 院校层次分布使用学校类别字典
- **WHEN** 系统生成院校层次分布图
- **THEN** 系统 MUST 基于在校大学生记录中的学校官方编码关联学校主数据
- **AND** 系统 MUST 使用学校类别字典中的类别值作为统计分组
- **AND** 系统 MUST 以环状图展示各学校类别的人数和占比

#### Scenario: 专业类别分布使用 14 大类专业类别
- **WHEN** 系统生成专业类别分布图
- **THEN** 系统 MUST 基于在校大学生记录中的专业官方编码关联专业主数据
- **AND** 系统 MUST 通过专业主数据进一步反查专业所属的专业分类作为统计分组
- **AND** 系统 MUST 不依赖青年信息表中单独保存的专业分类字段
- **AND** 系统 MUST 以竖向柱状图展示各专业类别的人数和占比

#### Scenario: 民族分类占比以饼图展示
- **WHEN** 系统生成民族分类占比图
- **THEN** 系统 MUST 使用在校大学生记录中的民族字段作为统计分组
- **AND** 系统 MUST 以饼图展示各民族的人数和占比

#### Scenario: 学历分布以横向柱状图展示
- **WHEN** 系统生成学历分布图
- **THEN** 系统 MUST 使用在校大学生记录中的学历字段作为统计分组
- **AND** 系统 MUST 以横向柱状图展示各学历的人数和占比

#### Scenario: 性别分布以左右对比条形图展示
- **WHEN** 系统生成性别分布图
- **THEN** 系统 MUST 使用在校大学生记录中的性别字段作为统计分组
- **AND** 系统 MUST 以左右对比条形图展示各性别的人数和占比

#### Scenario: 海东籍顶尖高校分析按学校标签动态生成
- **WHEN** 系统生成海东籍顶尖高校专题分析
- **THEN** 系统 MUST 基于海东籍在校大学生记录中的学校官方编码关联学校主数据和学校标签生成图表
- **AND** 系统 MUST 按学校标签数量动态输出对应数量的分析图
- **AND** 每张图的标题 MUST 直接使用对应学校标签名称
- **AND** 每张图 MUST 以环状图展示命中该标签高校的海东籍学生人数和占比

#### Scenario: 海东籍按籍贯区域判定
- **WHEN** 系统识别海东籍在校大学生
- **THEN** 系统 MUST 使用在校大学生记录中的籍贯区域编码作为判定依据
- **AND** 系统 MUST 仅将籍贯隶属于海东市的记录纳入海东籍专题分析

#### Scenario: 标签专题区域支持自适应换行
- **WHEN** 学校标签数量发生变化
- **THEN** 系统 MUST 使海东籍顶尖高校专题图表区域支持自适应换行展示
- **AND** 系统 MUST 保持各标签图在标签较少和较多时均可读

#### Scenario: 分析数据来自业务数据聚合
- **WHEN** 分析页请求图表数据
- **THEN** 系统 MUST 基于青年信息表、学校主数据、专业主数据和学校标签等业务数据实时聚合生成统计结果
- **AND** 系统 MUST 不依赖手工维护的静态统计数据
