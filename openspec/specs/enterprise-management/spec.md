## MODIFIED Requirements

### Requirement: 企业信息表单维护
系统 MUST 允许管理员通过统一管理列表与右侧抽屉表单流程对企业信息执行新增、编辑、查看、查询和删除操作，并在录入和查询过程中使用受控企业字典与三级区域代码；维护流程必须以更接近目标设计稿的列表布局和抽屉化表单视觉呈现。企业新增和编辑表单中的“所在地区”控件 MUST 与其他常规表单字段使用一致的列宽规则，不得长期以明显更长的整行控件破坏表单对齐。

#### Scenario: 新增企业信息
- **WHEN** 管理员在企业信息列表页点击新增并提交包含必填企业字段的合法企业表单
- **THEN** 系统保存企业信息
- **AND** 系统在企业列表中展示该记录

#### Scenario: 编辑企业信息
- **WHEN** 管理员在企业信息列表页点击编辑并修改一条已存在的企业信息后提交合法数据
- **THEN** 系统保存修改结果
- **AND** 系统在当前企业列表中展示更新后的企业信息

#### Scenario: 企业表单通过右侧抽屉承载
- **WHEN** 管理员在企业信息列表页点击新增或编辑
- **THEN** 系统必须在当前页面右侧滑出抽屉面板承载企业表单，并以更接近目标设计稿的布局展示表单字段、操作按钮和辅助信息
- **AND** 系统不得通过整页跳转打开独立表单页

#### Scenario: 企业表单使用受控字典
- **WHEN** 管理员新增或编辑一条企业信息
- **THEN** 系统必须对企业规模、企业性质和企业行业提供受控选项
- **AND** 系统不得要求管理员以自由文本方式录入这些字段

#### Scenario: 企业表单使用三级区域
- **WHEN** 管理员新增或编辑一条企业信息并填写所在地区
- **THEN** 系统必须提供省、市、区（县）级联选择
- **AND** 系统必须仅保存所在地区的区域代码组合

#### Scenario: 企业表单中的区域控件宽度与其他字段一致
- **WHEN** 管理员打开企业信息新增或编辑表单
- **THEN** 所在地区控件 MUST 与其他常规字段使用一致的列宽规则
- **AND** 该区域控件在桌面端不得明显长于相邻的普通输入框或下拉框

### Requirement: 企业信息仅支持表单录入
系统 MUST 提供与青年信息库一致的企业列表筛选与分页交互能力，并且第一阶段不得要求或提供企业 Excel 导入能力；企业列表页必须以统一视觉风格呈现筛选栏、操作按钮区和企业信息表格。企业信息列表页 MUST 将企业性质排在行业前面展示，并且业务表格列 MUST 不再显示企业规模。

#### Scenario: 按条件筛选企业列表
- **WHEN** 管理员通过企业名称、企业行业、企业性质、企业规模、所在地区等支持字段查询企业信息
- **THEN** 系统返回符合条件的企业记录
- **AND** 系统以统一风格刷新企业列表

#### Scenario: 企业列表使用统一布局与分页交互
- **WHEN** 管理员进入企业信息维护页面
- **THEN** 系统必须在页面上方展示查询筛选区
- **AND** 系统必须在查询区域下方右侧展示新增、删除等操作区
- **AND** 系统必须提供页号切换、每页条数切换与局部刷新能力

#### Scenario: 企业列表按新的业务字段优先级展示
- **WHEN** 管理员查看企业信息列表表格
- **THEN** 系统 MUST 在表格中将“企业性质”列排在“行业”列前面
- **AND** 系统 MUST 不再渲染“企业规模”列

#### Scenario: 删除企业前要求确认
- **WHEN** 管理员删除一条企业信息
- **THEN** 系统必须先展示平台风格的确认模态框
- **AND** 仅当管理员确认后系统才执行删除

#### Scenario: 企业模块不提供导入入口
- **WHEN** 管理员进入企业信息维护页面
- **THEN** 系统仅提供表单新增和编辑流程
- **AND** 系统不提供 Excel 导入入口

## ADDED Requirements

### Requirement: 企业信息删除保留软删除语义
系统 MUST 继续将企业信息视为可恢复的业务内容数据。管理员删除企业信息时，系统 MUST 通过软删除隐藏该企业，并在默认列表、查询和企业选择链路中排除已删除企业。

#### Scenario: 删除企业后从企业列表中移除
- **WHEN** 管理员删除一条企业信息
- **THEN** 系统 MUST 将该企业记录标记为已删除而不是物理移除
- **AND** 该企业 MUST 不再出现在默认企业列表和默认搜索结果中

#### Scenario: 企业选择链路忽略已软删除企业
- **WHEN** 招聘岗位维护或其他业务流程需要选择企业
- **THEN** 系统 MUST 不向管理员展示已软删除的企业
- **AND** 默认企业选项与联表查询结果 MUST 仅使用未删除企业

### Requirement: 企业信息支持人工排序字段
系统 MUST 在企业信息维护链路中提供一个名为“排序”的可空整数型字段。管理员 MUST 在新增与编辑企业信息时填写或清空该字段，系统 MUST 在企业信息详情中展示当前排序值，并保持详情页只读不直接编辑该字段。

#### Scenario: 在新增或编辑时维护排序值
- **WHEN** 管理员在企业信息新增或编辑表单中填写一个整数排序值或留空提交
- **THEN** 系统 MUST 保存该排序值或空值
- **AND** 系统 MUST 允许该字段不填

#### Scenario: 在详情中查看排序值
- **WHEN** 管理员打开一条企业信息详情
- **THEN** 系统 MUST 展示该记录当前保存的“排序”值
- **AND** 系统 MUST 不在详情页提供直接编辑该字段的交互

### Requirement: 企业信息列表按人工排序优先展示
系统 MUST 在企业信息列表中优先按人工排序字段展示记录。已填写“排序”的记录 MUST 排在未填写记录之前，并按排序值升序排列；当多个记录的排序值相同时，系统 MUST 继续按 `id desc` 排序；未填写排序值的记录 MUST 继续按现有 `id desc` 规则排序。企业信息列表页 MUST 不再显示“序号”列。

#### Scenario: 已填写排序值的记录优先显示
- **WHEN** 企业信息列表同时存在已填写排序值和未填写排序值的记录
- **THEN** 系统 MUST 先展示所有已填写排序值的记录
- **AND** 这些记录 MUST 按排序值从小到大排列

#### Scenario: 排序值相同或为空时继续使用默认顺序
- **WHEN** 多条企业信息的排序值相同，或这些记录都未填写排序值
- **THEN** 系统 MUST 在对应分组内继续按 `id desc` 排列

#### Scenario: 列表页不再显示序号列
- **WHEN** 管理员查看企业信息列表
- **THEN** 系统 MUST 不再渲染“序号”列
- **AND** 系统 MUST 保留现有分页信息与其他业务列
