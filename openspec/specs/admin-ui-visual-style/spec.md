## ADDED Requirements

### Requirement: 后台统一视觉壳子
系统必须为后台管理端提供统一的驾驶舱风格页面壳子，覆盖侧边导航、顶部标题区、内容面板和全局主题配色，使管理员在不同模块间切换时保持一致的视觉体验；该页面壳子 MUST 在共享顶栏承担主标题表达的前提下，减少页面内部重复的大标题卡、说明卡和嵌套容器，保持更紧凑、规整的工作区层级。

#### Scenario: 打开任一后台业务页面
- **WHEN** 已登录管理员进入首页、青年信息、企业信息、招聘信息、政策管理或分析页面
- **THEN** 系统展示统一的深色驾驶舱风格页面壳子，包括左侧导航、顶部标题区、统一面板边框和一致的页面背景层次
- **AND** 页面内部 MUST 避免重复表达同一页面标题或说明信息，优先将主要注意力留给当前工作区内容

### Requirement: 后台通用组件风格一致
系统必须对按钮、筛选栏、输入框、表格、提示信息和操作区采用统一的视觉规范，避免不同业务页面之间出现明显割裂的样式表现；这些通用组件 MUST 以更紧凑的间距、稳定的对齐关系和一致的密度呈现，减少不必要的大圆角、大阴影、大留白和重复卡片层级。

#### Scenario: 查看不同模块的列表和表单页面
- **WHEN** 管理员在不同业务模块之间切换并查看列表页、表单页或详情页
- **THEN** 系统以统一的组件风格呈现筛选区、操作按钮、表格、表单控件和提示信息，并保持一致的间距、描边、高亮和交互反馈
- **AND** 列表页中的筛选、操作、表格和分页 MUST 形成紧凑、整齐的工作台结构，而不是被分散在多个装饰性区块中

### Requirement: 后台筛选控件保持一致的空态与下拉展示
系统 MUST 让后台列表页中的查询筛选控件在初次加载和局部刷新后保持一致的空态表达、默认回显和下拉可读性。普通单选控件与区域级联控件即使使用不同实现，也 MUST 让管理员清晰区分“未选择/全部”和“已选择具体值”两种状态，且区域下拉面板 MUST 能完整展示较长的地区名称。

#### Scenario: 初次打开列表页时展示统一空态
- **WHEN** 管理员首次打开任一带查询筛选栏的后台列表页，且某个普通下拉或区域筛选未填写值
- **THEN** 系统 MUST 以占位提示样式展示该筛选控件
- **AND** 系统不得把“全部”或占位文案显示成与真实已选值相同的主文本颜色

#### Scenario: 局部刷新后保留空态语义
- **WHEN** 管理员执行查询、分页或重置并触发列表区域局部刷新，且某个筛选条件仍然为空
- **THEN** 系统 MUST 在刷新后继续将该筛选控件显示为占位提示状态
- **AND** 系统不得因为刷新后回退到原生控件而把空条件显示成普通已选值文本

#### Scenario: 区域下拉完整展示长名称
- **WHEN** 管理员展开后台列表页中的区域选择下拉，且候选地区名称或路径较长
- **THEN** 系统 MUST 让下拉面板以足够宽的展示区域显示候选项
- **AND** 系统不得将候选地区名称截断到仅剩部分字符而影响辨认

### Requirement: 后台页面使用正式业务化文案
系统 MUST 在后台首页、列表页、表单页、详情页、分析页、系统页和登录页使用正式、稳定、面向业务的展示文案，不得展示开发状态说明、设计说明、接口流程提示或演示占位词。

#### Scenario: 打开任一后台业务页面
- **WHEN** 管理员进入首页、青年信息、企业、招聘、政策、数据分析或系统设置页面
- **THEN** 系统展示的页面标题、副标题、说明文字和按钮文案 MUST 面向正式业务场景
- **AND** 系统不得展示“功能已打通”“保留现有流程”“与设计稿一致”等开发说明性文字

#### Scenario: 页面存在静态回退文案
- **WHEN** 模板中需要为列表、卡片、表格或标题提供静态回退文案
- **THEN** 系统 MUST 使用正式业务化回退文本或空状态文本
- **AND** 系统不得展示“示例企业”“示例政策”“张三”“Java开发工程师”等演示占位词

### Requirement: 后台导航避免重复层级入口
系统 MUST 优先使用一级菜单配合页面内工作台或页签展示业务内容，不得在同一模块中同时使用侧边子菜单和页面内页签表达同一组入口，造成重复导航。

#### Scenario: 打开使用页签切换的业务模块
- **WHEN** 管理员进入青年信息、数据分析或系统设置等带有页签或工作台切换区域的模块
- **THEN** 系统 MUST 通过统一的页面内切换区域承载该模块的细分内容
- **AND** 系统不得在左侧继续强调同一组重复子入口

### Requirement: 后台看板和分析页采用规整网格
系统 MUST 将首页和数据分析页组织为规整、可响应的看板网格，使地图、统计数字、图表和摘要列表在大屏和常规桌面下都保持明确对齐关系；页面不得依赖多层嵌套卡片和重复标题制造视觉层次。

#### Scenario: 打开首页或分析页
- **WHEN** 管理员进入首页或任一数据分析页面
- **THEN** 系统 MUST 通过规则网格展示地图、统计卡、图表和摘要数据
- **AND** 页面中的主要视觉焦点 MUST 是地图、数字和图表本身，而不是额外的装饰性标题卡或说明卡

### Requirement: 后台抽屉呈现为工作侧板
系统 MUST 将右侧抽屉页面呈现为统一的工作侧板，确保头部、正文和底部操作区之间的层级关系明确，并在桌面端和移动端都保持稳定的可用性；抽屉不得再次模拟一个完整页面壳子。

#### Scenario: 在列表页中打开新增、编辑或详情抽屉
- **WHEN** 管理员从任一后台列表页打开新增、编辑或详情内容
- **THEN** 系统 MUST 以紧凑、规整的侧板结构展示标题、表单或详情信息以及底部操作区
- **AND** 抽屉内部 MUST 避免重复的页面级标题壳子、过大的留白和不整齐的表单排布
