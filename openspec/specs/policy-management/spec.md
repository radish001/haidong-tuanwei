## ADDED Requirements

### Requirement: 政策信息维护
系统必须允许管理员以与青年信息库一致的列表布局、删除确认和右侧抽屉交互对政策信息执行新增、编辑、查看、删除和发布状态管理，并支持富文本正文内容；政策列表、详情和编辑呈现必须以更接近目标设计稿的布局和视觉层次展示筛选栏、操作区、状态列与内容面板。

#### Scenario: 保存政策草稿
- **WHEN** 管理员在政策列表页点击新增或编辑并提交包含标题、基础元数据和富文本正文的政策表单
- **THEN** 系统保存该政策记录
- **AND** 系统保留可再次编辑的富文本正文内容

#### Scenario: 变更政策发布状态
- **WHEN** 管理员对一条政策执行发布或取消发布操作
- **THEN** 系统更新该政策的发布状态
- **AND** 系统在政策列表中反映最新状态

#### Scenario: 政策列表使用统一布局与分页交互
- **WHEN** 管理员进入政策管理列表页
- **THEN** 系统必须在页面上方展示查询筛选区，并以统一风格呈现筛选栏、操作按钮区、政策列表和状态列，使页面整体更接近目标设计稿
- **AND** 系统必须在查询区域下方右侧展示新增、删除等操作区
- **AND** 系统必须提供页号切换、每页条数切换与局部刷新能力

#### Scenario: 政策详情与编辑通过右侧抽屉打开
- **WHEN** 管理员在政策列表页点击详情、新增或编辑
- **THEN** 系统必须在当前页面右侧滑出抽屉面板承载对应内容，并以更接近目标设计稿的内容面板与视觉层次呈现政策信息
- **AND** 系统不得通过整页跳转进入独立详情页或编辑页

#### Scenario: 删除政策前要求确认
- **WHEN** 管理员删除一条政策信息
- **THEN** 系统必须先展示平台风格的确认模态框
- **AND** 仅当管理员确认后系统才执行删除

### Requirement: 政策信息删除保留软删除语义
系统 MUST 继续将政策信息视为可恢复的业务内容数据。管理员删除政策信息时，系统 MUST 通过软删除隐藏记录，并在默认列表、详情入口和状态管理链路中排除该记录。

#### Scenario: 删除政策后从默认列表中移除
- **WHEN** 管理员删除一条政策信息
- **THEN** 系统 MUST 将该政策记录标记为已删除而不是物理移除
- **AND** 该政策 MUST 不再出现在默认政策列表和默认分页结果中

#### Scenario: 已软删除政策不再参与默认状态管理
- **WHEN** 管理员查看政策列表或执行默认状态筛选
- **THEN** 系统 MUST 忽略已软删除政策
- **AND** 已删除政策 MUST 不再作为可发布、可编辑或可查看的默认记录出现

### Requirement: 政策信息支持人工排序字段
系统 MUST 在政策信息维护链路中提供一个名为“排序”的可空整数型字段。管理员 MUST 在新增与编辑政策信息时填写或清空该字段，系统 MUST 在政策详情中展示当前排序值，并保持详情页只读不直接编辑该字段。

#### Scenario: 在新增或编辑时维护排序值
- **WHEN** 管理员在政策新增或编辑表单中填写一个整数排序值或留空提交
- **THEN** 系统 MUST 保存该排序值或空值
- **AND** 系统 MUST 允许该字段不填

#### Scenario: 在详情中查看排序值
- **WHEN** 管理员打开一条政策详情
- **THEN** 系统 MUST 展示该记录当前保存的“排序”值
- **AND** 系统 MUST 不在详情页提供直接编辑该字段的交互

### Requirement: 政策列表按人工排序优先展示
系统 MUST 在政策列表中优先按人工排序字段展示记录。已填写“排序”的记录 MUST 排在未填写记录之前，并按排序值升序排列；当多个记录的排序值相同时，系统 MUST 继续按 `id desc` 排序；未填写排序值的记录 MUST 继续按现有 `id desc` 规则排序。政策列表页 MUST 不再显示“序号”列。

#### Scenario: 已填写排序值的记录优先显示
- **WHEN** 政策列表同时存在已填写排序值和未填写排序值的记录
- **THEN** 系统 MUST 先展示所有已填写排序值的记录
- **AND** 这些记录 MUST 按排序值从小到大排列

#### Scenario: 排序值相同或为空时继续使用默认顺序
- **WHEN** 多条政策的排序值相同，或这些记录都未填写排序值
- **THEN** 系统 MUST 在对应分组内继续按 `id desc` 排列

#### Scenario: 列表页不再显示序号列
- **WHEN** 管理员查看政策列表
- **THEN** 系统 MUST 不再渲染“序号”列
- **AND** 系统 MUST 保留现有分页信息与其他业务列
